一、概述

1、C的起源

1972年，贝尔实验室的丹尼斯·里奇和肯·汤普逊在开发UNIX操作系统时设计了C语言。（在B语言基础上)

C语言设计的初衷是将其作为程序员使用的一种编程工具。

2、选择C语言的理由

优点：

1）设计特性

C语言的设计理念让用户能轻松地完成自顶向下的规划、结构化编程和模块化设计。

用C编写的程序更易懂、更可靠。

2）高效性

C是高效的语言。C程序紧凑、运行速度快。具有汇编语言的微调控制能力。

3）可移植性

在一种系统中编写的C程序几乎不用修改就能在其他系统上运行。

4）强大而灵活

5）面向程序员

缺点：

C语言使用指针，容易出现难以发现的错误。

3、C语言的应用范围

嵌入式系统，操作系统（Linux）等

4、语言标准

1989 年 ANSI C 标准 定义了C语言和C标准库 （通常称作C89标准或C90标准）

1999 年 C99

2011 年 C11，但是并不是所有编译器都支持。

每一代标准都有一些不同,但差别不大。

3、位、字节和字

位、字节和字都是用于描述计算机数据单元或者存储单元的单位，在这里特指存储单元。

最小的存储单元是比特位（bit），比特位存储的是数字0和1，可以将它们看作是开关中的开和关。比特位是计算机中最基本的存储单元和最基本的内存构建块。

字节（byte）是常用的计算机存储单元。几乎对于所有的机器，1字节为8位，即1byte = 8 bit。

字（word）是设计计算机时给定的自然存储单位。对于8为微型计算机，1字长只有8位，后来的16位、32位、到目前的64位，字长越长，数据转移越快，允许的内存访问也就越多。

4.#define和const

在C语言中，用#define定义一个常量，这个方式叫预处理器，它的作用是在编译时将源码中的常量名替换为值，这种方式叫编译时替换，这种方式定义的常量叫明示常量也叫符号常量。

我们约定俗成的常量名命名方法是全部大写。

在C90时，C语言添加了一个关键字–“const”。

它的作用也是定义常量，那么它究竟和#define有什么区别呢？

define与const都与常量有关。

define是宏定义，它可以把一个标识符变成一个常量，而const是一个修饰符，它可以修饰一个变量为常类型，使其不可被修改。

本质上两者不同，define修饰出来的是常量。并且是真常量。而const修饰的是假常量，它本质还是变量，只不过是不能修改值。

为什么说define修饰出来的是真常量呢？

因为，define是宏定义，是宏替换！意味着程序在编译前期会进行文本替换，例如代码define PI 3.14，编译器会将代码中所有出现PI的地方全部用3.14替换，然后在进行编译，所以本质它就是一个常量！
至于const，const一般修饰于定义变量的前面，比如const int a=10，意味着a就一直是10，是一个不可修改的变量。

往后给a重新赋值都会引发错误。

这就是const修饰之后的结果，但请注意这里不可修改的原因其实是编译器在做检查，检查是否有修改这块内存上的值，一旦有就会报错！所以从根本上说，const修饰的变量依然是一个变量！只不过编译器不让修改而已！

1）编译器处理方式不同
define 宏是在预处理阶段展开。
const 是编译运行阶段使用。
2）类型和安全检查不同
define 宏没有类型，不做任何类型检查，仅仅是展开。
const 有具体的类型，在编译阶段会执行类型检查。
3）存储方式不同
define 宏仅仅是展开，有多少地方使用，就展开多少次，不会分配内存（宏定义不分配内存，变量定义分配内存）。
const 会在内存中分配（可以是堆中也可以是栈中）
4）const可以节省空间，避免不必要的内存分配。
const 定义常变量从汇编的角度来看，只是给出了对应的内存地址，而不是像define一样给出的是立即数，所以，const定义的常变量在程序运行过程中只有一份拷贝（因为是全局的只读变量，存在静态存储区），而define定义的常变量在内存中有若干个拷贝。
5）编译器通常不为普通const常变量分配存储空间，而是将它们保存在符号表中，这使得它成为一个编译期间的常量，没有了存储与读内存的操作，使得它的效率也很好。
6）宏替换只做替换，不做计算，不做表达式求解。
7）宏预编译时就替换了，程序运行时，并不分配内存。

总结

类别 define const
何时处理 编译期间进行宏替换 编译期间检查是否有修改
谁来执行 由系统判断是否被修改 由编译器限制
存储位置 数据常量在内存只读区 变量在哪里就在哪里

但是最最重要的是const有类型检查，更推荐使用。

2.缓冲区

在某些老式的系统中，当用户输入时，系统会在回显用户输入的字符后直接打印该字符，比如输入“hello”就会在屏幕上出现“hheelllloo”的字样，这种输入叫做“无缓冲输入”也叫“直接输入”。

随着时代的发展，这种输入方式已经消失在了历史的尘埃之中，我们现在的系统都是等待用户按下回车键后再开始打印，这种输入方式叫做“缓冲输入”，用户的输入被暂存在一个叫“缓冲区”（buffer）的临时性存储区域内。按下回车后，程序才可以使用用户输入的内容。

这种方式将用户输入的若干字符当作一个块进行传输，这样节约时间和系统资源。其次，在缓冲区内可以低成本的修正错误，保证在按下回车时，程序使用的是正确的输入。

但是，也并非是所有的程序都要缓冲输入，比如打游戏时，我们就必须实时的执行相应指令，而不是输入一对操作之后再依次执行。

缓冲执行分为完全缓冲I/O和行缓冲I/O。完全缓冲IO指的是缓冲区被填满后才书信缓冲区（内容发送至目的地），通常出现在文件输入中。缓冲区的大小取决于系统，常见的大小是512字节和4096字节。行缓冲I/O指的是在出现换行符时刷新缓冲区。键盘输入通常是行缓冲I/O，在按下回车键后才刷新缓冲区。

3.文件、流和键盘输入

文件（file）是存储器中存储信息的区域。通常，文件保存在永久存储器中，比如：硬盘、U盘或者光盘。C语言中有许多用于打开、读取、写入和关闭文件的函数，在较低层面上，C可以使用主机操作系统的基本文件工具操作文件，这些直接调用操作系统的函数被称为底层I/O（low-level I/O）。但实际上，因为操作系统各不相同，所以没办法为底层I/O创建标准库，因此，C中有了处理文件的标准I/O包（standard I/O package）。标准IO包得以让开发者无视各个系统之间的差异，使用同一的标准来操作。

从概念上来看，C语言处理的是流而非直接处理文件。流（Stream）是一个实际输入或输出映射的理想化数据流。打开文件就是将流与文件相关联，并通过读写流来实现读写文件。

我们常用的键盘输入是标准输入流，即“stdin”；与之相对的屏幕输出叫标准输出流，即“stdout”。getchar、putchar、printf、scanf都是标准IO包的成员，处理这两个流。

4.尾递归

如果一个函数中所有递归形式的调用都出现在函数的末尾，我们称这个递归函数是尾递归的。当递归调用是整个函数体中最后执行的语句且它的返回值不属于表达式的一部分时，这个递归调用就是尾递归。尾递归函数的特点是在回归过程中不用做任何操作，这个特性很重要，因为大多数现代的编译器会利用这种特点自动生成优化的代码。

3.gets、fgets、gets_s和s_gets

1）有缺陷的gets

如果不算上我们[^\n]的骚操作，scanf仅能读取一个单词，然后在遇到空格、换行符和结尾时停止读取。但很多时候我们需要读取一整行输入，这时我们会用gets来代替scanf。然而，我们之前说过gets（还有scanf）是不安全的，它们在无法确定数组是否能装下读入的数据的情况下，很可能会造成缓冲区溢出。

2.如果fgets（）读到一个换行符，会把它储存在字符串中。这跟gets（）不太一样，gets（）会丢弃换行符。

3.fgets（）的第三个参数知名要读入的文件。。如果读入从键盘输入的数据，则使用stdin。

十一、指针再谈

我们知道了指针变量是存储地址的一种变量，也知道了指针和数组之间的关系。那么，指针到底是啥，又该怎么用呢？

想要理解C语言的指针，就要从内存开始。

1.内存、地址和指针

计算机中所有的数据都必须放在内存中，不同类型的数据占用的字节数不一样，例如 int 占用 4 个字节，char 占用 1 个字节。为了正确地访问这些数据，必须为每个字节都编上号码，就像门牌号、身份证号一样，每个字节的编号是唯一的，根据编号可以准确地找到某个字节。

我们将内存中字节的编号称为地址（Address）或指针（Pointer）。地址从 0 开始依次增加，对于 32 位环境，程序能够使用的内存为 4GB，最小的地址为 0，最大的地址为 0XFFFFFFFF。

2.一切皆地址

C语言用变量来存储数据，用函数来定义一段可以重复使用的代码，它们最终都要放到内存中才能供 CPU 使用。

数据和代码都以二进制的形式存储在内存中，计算机无法从格式上区分某块内存到底存储的是数据还是代码。当程序被加载到内存后，操作系统会给不同的内存块指定不同的权限，拥有读取和执行权限的内存块就是代码，而拥有读取和写入权限（也可能只有读取权限）的内存块就是数据。

CPU 只能通过地址来取得内存中的代码和数据，程序在执行过程中会告知 CPU 要执行的代码以及要读写的数据的地址。如果程序不小心出错，或者开发者有意为之，在 CPU 要写入数据时给它一个代码区域的地址，就会发生内存访问错误。这种内存访问错误会被硬件和操作系统拦截，强制程序崩溃，程序员没有挽救的机会。

注意：CPU 访问内存时需要的是地址，而不是变量名和函数名！变量名和函数名只是地址的一种助记符，当源文件被编译和链接成可执行程序后，它们都会被替换成地址。编译和链接过程的一项重要任务就是找到这些名称所对应的地址。

需要注意的是，虽然变量名、函数名、字符串名和数组名在本质上是一样的，它们都是地址的助记符，但在编写代码的过程中，我们认为变量名表示的是数据本身，而函数名、字符串名和数组名表示的是代码块或数据块的首地址。

3.指针变量

数据在内存中的地址也称为指针，如果一个变量存储了一份数据的指针，我们就称它为指针变量。

在C语言中，允许用一个变量来存放指针，这种变量称为指针变量。指针变量的值就是某份数据的地址，这样的一份数据可以是数组、字符串、函数，也可以是另外的一个普通变量或指针变量。

5.通过指针变量取得数据

因为指针存储了数据的地址，所以我们可以通过指针变量获取该地址上的数据，使用之前提到过的“*”。

假设 a 的地址是 0X1000，p 指向 a 后，p 本身的值也会变为 0X1000，*p 表示获取地址 0X1000 上的数据，也即变量 a 的值。从运行结果看，*p 和 a 是等价的。

但是，CPU 读写数据必须要知道数据在内存中的地址，普通变量和指针变量都是地址的助记符，虽然通过 *p 和 a 获取到的数据一样，但它们的运行过程稍有不同：a 只需要一次运算就能够取得数据，而 *p 要经过两次运算，多了一层“间接”。

程序被编译和链接后，a、p 被替换成相应的地址。使用 *p 的话，要先通过地址 0XF0A0 取得变量 p 本身的值，这个值是变量 a 的地址，然后再通过这个值取得变量 a 的数据，前后共有两次运算；而使用 a 的话，可以通过地址 0X1000 直接取得它的数据，只需要一步运算。

这也就是说，使用指针是间接获取数据，使用变量名是直接获取数据，前者比后者的代价要高。

如果一个指针指向了数组，我们就称它为数组指针（Array Pointer）。

我们虽然在数组名和指针地址上画了等号，但是记住，它们绝不是同一种东西。

数组在内存中只是数组元素的简单排列，没有开始和结束标志，在求数组的长度时不能使用sizeof(p) / sizeof(int)，因为 p 只是一个指向 int 类型的指针，编译器并不知道它指向的到底是一个整数还是一系列整数（数组），所以 sizeof(p) 求得的是 p 这个指针变量本身所占用的字节数，而不是整个数组占用的字节数。

也就是说，根据数组指针不能逆推出整个数组元素的个数，以及数组从哪里开始、到哪里结束等信息。不像字符串，数组本身也没有特定的结束标志，如果不知道数组的长度，那么就无法遍历整个数组。

但是数组指针可以通过指针的运算来遍历它们，因为如果一个指针变量 p 指向了数组的开头，那么 p+i 就指向数组的第 i 个元素；如果 p 指向了数组的第 n 个元素，那么 p+i 就是指向第 n+i 个元素；而不管 p 指向了数组的第几个元素，p+1 总是指向下一个元素，p-1 也总是指向上一个元素。

最后，我们再来看看假设 p 是指向数组 arr 中第 n 个元素的指针，那么 p++、++p、(*p)++ 分别是什么意思呢？

*p++ 等价于 *(p++)，表示先取得第 n 个元素的值，再将 p 指向下一个元素，上面已经进行了详细讲解。

*++p 等价于 *(++p)，会先进行 ++p 运算，使得 p 的值增加，指向下一个元素，整体上相当于 *(p+1)，所以会获得第 n+1 个数组元素的值。

(*p)++ 就非常简单了，会先取得第 n 个元素的值，再对该元素的值加 1。假设 p 指向第 0 个元素，并且第 0 个元素的值为 99，执行完该语句后，第 0 个元素的值就会变为 100。

12.野指针和垂悬指针

野指针：访问一个已销毁或者访问受限的内存区域的指针，野指针不能判断是否为NULL来避免

垂悬指针：指针正常初始化，曾指向一个对象，该对象被销毁了，但是指针未制空，那么就成了悬空指针。

它们都是指向了一块随机的空间，不受程序控制。

野指针产生的原因：
1.指针定义时未被初始化：指针在被定义的时候，如果程序不对其进行初始化的话，它会随机指向一个区域，因为任意指针变量（除了static修饰的指针）它的默认值都是随机的

2.指针被释放时没有置空：我们在用malloc（）开辟空间的时候，要检查返回值是否为空，如果为空，则开辟失败；如果不为空，则指针指向的是开辟的内存空间的首地址。指针指向的内存空间在用free()和delete释放后，如果程序员没有对其进行置空或者其他赋值操作的话，就会成为一个野指针

3.指针操作超越变量作用域：不要返回指向栈内存的指针或者引用，因为栈内存在函数结束的时候会被释放。

野指针会导致一些奇奇怪怪的Bug，所以我们在定义和使用指针时要注意：

  • 1.初始化指针的时候将其置为nullptr，之后对其操作。
  • 2.释放指针的时候将其置为nullptr。

2.作用域

作用域描述程序中可访问标识符的区域。一个C变量的作用域可以是块作用域、函数作用域、函数原型作用域或者文件作用域。

1）块作用域

块是一对花括号括起来的区域，定义在块中的变量就是块级变量，块作用域变量的可见范围是从定义处到包含该定义的块的结尾。形参虽然定义在块的左边，也就是圆括号内，但是它同样属于该块作用域。

2）函数作用域（无意义）

函数作用域仅用于goto语句的标签，因为现代C程序不建议使用goto，所以知道有这么个东西就行。

3）函数原型作用域

函数原型作用域是从形参定义到函数原型声明结束，这也就是说，函数原型中的形参，变量名是不重要的，变量类型才重要，除非是变长数组，否则可以没有变量名。

4）文件作用域

变量定义在函数的外面就具有文件作用域，它的范围是从定义处到该定义所在文件的末尾。有点类似于其他一些语言的全局作用域，但又不完全一样（后面会有提到）。

5）翻译单元和文件

多个文件在编译器中可能以一个文件的形式出现，

比如C源文件包含若干C头文件，这些物理文件在C预处理时会替换#include语句，所以实际上C源文件和所有它包含的头文件都可以看成是一个文件，这个文件叫“翻译单元”。实际上，一个文件作用域的变量，其可见范围是整个翻译单元。

如果一个程序由多个源文件组成，那么每个源文件及其各自包含的头文件都成为一个翻译单元，然后这些翻译单元组成程序。

3.链接

C变量有3种连接属性：外部链接、内部链接或者无链接。

具有块作用域、函数作用域或函数原型作用域的变量都是无链接变量。这意味着这些变量属于定义它们的块、函数或者原型私有。

具有文件作用域的变量可以说外部或者内部链接。外部链接可以在多文件程序中使用，内部链接只能在一个翻译单元内使用。

既有内部链接的文件作用域叫做文件作用域，具有外部链接的文件作用域叫全局作用域或者程序作用域。

4.存储期

存储器描述了通过标识符访问俺对象的生存期，C语言有4种存储期：静态存储期、线程存储期、自动存储期、动态分配存储期。

1）静态存储期

如果对象具有静态存储期，那么它在程序的执行期间一直存在。文件作用域变量具有静态存储期，对于文件作用域变量来说，关键字static表明了其链接属性，而非存储期。以static声明的文件作用域变量具有内部链接，但无论是内部链接还是外部链接，所有文件作用域变量都具有静态存储期。

2）线程存储期

线程存储期用于并发程序设计，程序执行可被分为多个线程。具有线程存储期的对象，从被声明时到线程结束一直存在。以关键字_Thread_local声明一个对象时，每个线程都要获得该变量的私有备份。

3）自动存储期

块作用域的变量通常具有自动存储期，程序进入定义定义这些变量的块时，为这些变量分配内存；退出这些变量的块时，释放刚刚为变量分配的内存。这就相当于把自动变量占用的内存是为一个可重复使用的暂存区。

4）动态分配存储期

比如变长数组，它们的存储期是从声明到块的末尾。

6.自动变量

自动变量具有自动存储期、块作用域且无链接。默认情况下，声明在块或者函数头种的任何变量都是自动存储类别。定义自动存储类别的变量可以显式的使用关键字auto。

但是因为C和C++里，auto关键字的作用不一样，所以如果要写C、C++兼容的程序，最好还是不要这样用。

块作用域和无链接意味着只幼仔定义的块种才能访问到这个变量（除了作为参数传递给其他函数，这是特殊情况了）。另一个函数中可以定义同名变量，但是那是同一标识符、存储在不同内存位置的不同变量。

当块内部和外部变量同名时，优先使用内部的变量。

7.寄存器变量

变量通常存储在计算机内存中，但是我们可以利用关键字register将它存储在计算机最快的可用内存寄存器中，这就是寄存器存储类型变量。

但是，register并非是一种指令或者说命令，它更像是一种请求，因为并不是任何时候变量都能放在寄存器中，编译器会根据寄存器或者最快可用内存的数量来衡量你的请求–接受或者拒绝。当拒绝时，寄存器变量就会变成普通的自动变量，并且你不能对该变量使用地址运算符。

十三、文件输入/输出

1.文件定义

文件是在磁盘或固态硬盘上一段已经命名的存储区。

C语言将文件看成一系列连续的字节，每个字节都能单独的读取。并且提供两种文件模式：文本模式和二进制模式。

2.文本模式和二进制模式

首先需要区分文本内容、二进制内容、文本文件格式和二进制文件格式，以及文本模式和二进制模式。

所有文件都以二进制存储，但是，如果文件使用二进制编码的字符（例如ASCII或者Unicode）表示文本，该文件就是文本文件，其中包含文本内容；如果文件使用二进制值代表机器语言代码或者数值数据或者图片编码或者音视频编码，该文件就是二进制文件，其中包含二进制内容。

为了规范文本文件的处理，C语言提供两种访问文件的途径：文本模式和二进制模式。二进制模式中，程序可以访问文件的每一个字节；而文本模式中，程序打印的内容和文件的实际内容不同。

当然，虽然C语言提供了两种模式，但是在UNIX和LINUX中，这两种模式的实现完全相同。

8.将结构体作为参数

现代C语言支持三种结构体作为参数的情况。

1）传递结构体成员

传递结构体成员就是定义普通的函数，将结构体的成员作为参数传递。

2）传递结构体地址（指针）

传递结构体地址就是将函数的参数设为指针，然后将结构体的地址传递进去，这样函数的参数就成了一个指向结构体的指针变量。

3）传递结构体

传递结构体就是直接将结构体做参数传递进去。

4）该如何选择？

我们应该选择传递结构或者传递结构指针呢？它们各有优缺点。

传递结构指针对于老代码的兼容性好，因为传递结构是后来才有的能力；其次就是传递结构指针执行速度快，它只需要传递一个地址就好，而不是整个结构的大量数据。但是，传递结构指针无法保护数据，被调用函数的某些操作可能会影响源数据。当然，现在我们可以使用const解决这个问题，限定为const的参数遭到修改时，编译器会报错。

传递结构处理的是源数据的副本，这保护了原始数据。同时传递结构使得代码更加清晰。但是传递结构兼容性很差，同时会浪费大量的时间和空间。

通常，处理大型数据时，为了追求效率我们会传递指针同时使用const限定。而处理小型数据时，我们会直接处理结构成员。
